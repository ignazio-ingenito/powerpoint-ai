# Data Pipeline Blueprint

Cartella progetto per la presentazione sulla blueprint di una data pipeline riusabile su piu' iniziative: Kiron CDG, Internal CDG e ProSIGNAL.

## Struttura

| Path | Contenuto |
|---|---|
| `drafts/` | Intake, inventario fonti, repo-to-deck brief, storyline e creative handoff |
| `prompts/` | Prompt usati per recupero materiale, analisi repository o generazione deck |
| `source-materials/` | Materiali sorgente raccolti da repo, documenti, deck e file di progetto |
| `visual-references/` | Reference visive specifiche della presentazione |
| `generated-assets/` | Asset o script generati per questa presentazione |
| `attempts/` | Tentativi, export intermedi e output non finali |

## Obiettivo Iniziale

Preparare una presentazione executive/technical blueprint che spieghi come costruire una data pipeline riusabile, partendo dalle evidenze dei progetti CDG e rendendo esplicito cosa puo' essere generalizzato per Kiron CDG, Internal CDG e ProSIGNAL.

## Stato

- Scaffolding creato.
- Primo materiale locale raccolto da `cdg-data-pipeline`, `cdg-data-pipeline-docs`, `tools` e Obsidian TXT Novigo.
- Sintesi iniziale dei casi CDG/ProSignal disponibile in `drafts/03-use-case-synthesis.md`.
- Kiron resta un beneficiario target senza materiale locale specifico recuperato.
- Nessun PowerPoint generato.
- Prossimo passo: completare il dossier `Repo to Deck Brief` e classificare i gap bloccanti prima della storyline.
