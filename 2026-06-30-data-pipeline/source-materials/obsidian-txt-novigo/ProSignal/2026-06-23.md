
# Requisiti

- Possibilità di modificare i tracciati dei file (campi, tipi, ecc...)
- File più grande 7G (in aumento) - Pagamenti (carte, )
- Qualunque tipo di file di testo (posizionale), quelli di Lease e Conto arancio hanno un tracciato particolare (da verificare per le versioni successive)
- Come prima fase verrà fatto con un upload manuale
- In alcuni file ci sono dei record di testa e coda che possono essere usate per verifiche di integrità. Per gli altri viene fatto un controllo di scostamento andamentale.
- ProSIGNAL deve anche funzionare da editor puro di testo quindi poter aprire il file come file di testo puro con la possibilità di column mode.
- Possibilità di ordinare e filtrare i file.
- I file di ProSIGNAL dovranno sempre essere in formato DOS e non UNIX

# Prima fase di analisi
Viene creato un file excel con grandezze aggregate (per ogni forma tecnica) come 
- Quantità
- Importo 
- Numero operazioni

A seconda della forma tecnica viene fatto un controllo di scostamento andamentale di una delle tre grandezze rispetto ai mesi precedenti.

Per questo calcolo vengono usati solo alcuni campi, non tutti.

# File decadali

I file decadali hanno una maggiore complessità perchè va fatto un controllo incrociato tra più file per tutte le forme tecniche 

Per esempo viene incrociato l'input con il bilancio, o con elaborazioni fatte all'esterno.

Il file da fornire a Banca d'Italia viene preparato in ING perchè se Nexen non produce l'ouput per BI ING manda il proprio per poi rettificare (se è il caso) quando il file di Nexen arriva.

Come prima ipotesi potremmo alimentare i file excel per le verifiche in modo da ridurre l'effort e renderlo più modificabile.
